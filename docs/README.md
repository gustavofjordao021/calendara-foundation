# Calendara feature specs

This folder holds **feature specs and design system docs** — PRDs for individual features, not product strategy.

For product strategy (vision, roadmap, competitive, market, GTM, growth) see [`../strategy/`](../strategy/README.md).

## Contents

- `DESIGN_SYSTEM.md` — visual language, tokens, components
- `FEATURE_AGENDA_EMAILS.md` — daily/weekly agenda email spec
- `FEATURE_BIRTHDAY_TRACKER.md`
- `FEATURE_FAMILY_COLORS.md`
- `FEATURE_HYGGE_AGENT_PHASE1.md` — Phase 1 agent build plan: Create · Query · Move (June 2026)
- `FEATURE_MEMORY_AND_ASSISTANT.md` — agent + Memory Lane product grouping (April 2026)
- `FEATURE_MONTH_VIEW.md`
- `FEATURE_ONBOARDING.md`
- `FEATURE_PRIVACY_CONTROLS.md`
- `FEATURE_REMINDERS.md`
- `FEATURE_SEARCH.md`
- `FEATURE_WEEK_VIEW.md`

## Adding a new spec

1. Use the pattern `FEATURE_<NAME>.md` for discoverability.
2. Link the spec from `../strategy/roadmap/product-roadmap-*.md` so it shows up in the "Ready to Build" index.
3. If the feature is strategy-defining (not just an implementation), also reference it from `../strategy/vision/vision-hygge.md` or the current roadmap.
